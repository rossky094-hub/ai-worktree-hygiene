# GitHub Discoverability Notes

This repository is packaged so people can find it through GitHub search and topic
browsing when they look for AI coding hygiene, git hygiene, dirty worktree cleanup,
phase-clean checkpoints, token waste reduction, agent workflows, subagent review,
versioned slice hygiene, and commit hygiene.

## Search Fields To Optimize

GitHub repository search supports matching across repository name, description,
README, and topics. The project therefore repeats important phrases naturally in:

- repository name: `ai-worktree-hygiene`
- short description: `Phase-clean checkpoint for AI coding agents: stop branch drift, reduce token waste, and keep worktrees, subagent diffs, and versioned slices reviewable.`
- README first screen
- topics
- examples and templates

## Natural Star Growth Strategy

Stars grow when a project is easy to understand, solves a narrow painful problem,
and gives contributors obvious next steps.

This repo supports that by:

- naming a problem users already feel: tests pass but the branch is not reviewable
- explaining that stronger agents drift faster without phase boundaries
- making the first useful action copyable
- including realistic before/after examples
- explaining why tracked clean is not phase clean
- covering subagent diff review and existing-chain gaps without becoming a platform
- separating implemented source from stale generated artifacts
- keeping versioned slices and authored artifacts linked to evidence
- providing issue templates for dirty worktree scenarios
- keeping the license simple
- avoiding internal project names and local paths
- linking to adjacent tools without claiming to replace them

## Launch Checklist

- Set GitHub description.
- Set topics.
- Add MIT license.
- Pin or share with a short demo screenshot or terminal transcript.
- Open one or two `good first issue` examples after launch.
- Keep README first screen focused on the pain and quick start.
